# Local Service Deploy Reference

This reference records the supported local deployment path for running Vela continuously on a user machine.

## User Level `systemd` Is The Canonical Continuous Runtime

- The governed API runs as `vela-api.service`. (2026-04-12)
  - It serves the local HTTP contract and keeps Vela reachable for n8n and operator calls.
- Warden patrol runs from `vela-patrol.service` on `vela-patrol.timer`. (2026-04-12)
  - The timer cadence is `02:00, 06:00, 10:00, 14:00, 18:00, 22:00`.
- The night cycle runs from `vela-night-cycle.service` on `vela-night-cycle.timer`. (2026-04-12)
  - The timer cadence is `03:00`.

## Install Path

- Unit directory source: `ops/deploy/systemd/`. (2026-04-12)
  - These files are copied into the user unit directory by the bootstrap script.
- Local user unit destination: `~/.config/systemd/user/`. (2026-04-12)
  - This keeps deployment local to the user rather than machine-global.
- Install helper: `ops/bootstrap/install-user-services.sh`. (2026-04-12)
  - The helper copies units, installs a local env template, and prints the enable commands.

## Runtime Defaults

- The service env template is `ops/deploy/systemd/vela.env.example`. (2026-04-12)
  - It defines `VELA_HOST`, `VELA_PORT`, `VELA_PATROL_ACTOR`, and `VELA_NIGHT_CYCLE_ACTOR`.
- The default patrol actor is `system`. (2026-04-12)
  - Scheduled patrol is a system-owned operation, not a human-issued one.
- The default night-cycle actor is `system`. (2026-04-12)
  - Scheduled night maintenance should enter through the same governed actor consistently.

## Governance Notes

- Continuous deployment does not bypass approval, validation, or event logging. (2026-04-12)
  - The service layer stays governed even when started automatically.
- The local service path is the supported continuous runtime for this machine. (2026-04-12)
  - Other deployment forms may exist later, but user services are the canonical local path today.
