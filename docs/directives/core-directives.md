# Core Directives

## This Document Names the Directives That All Implementations Must Respect
The system enforces SoT supremacy, single-writer discipline, role purity, reflection before mutation, human gates on sovereignty, validation before commit, one home with many pointers, complete event logging, pure cores with impure edges, narrative structure, conservative self-modification, and sequential interplay.

## This Document Clarifies That Behavior Matters More Than Slogans
Tests, validators, and runtime gates are expected to prove the directives through observed behavior rather than through wording alone.
